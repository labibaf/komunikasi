A) Apa ?
	1. Apa itu Display Manager (DM)?
          Dalam X Window System, manajer tampilan berjalan sebagai sebuah 
       program, yang memungkinkan dimulainya sesi di server X dari 
       komputer yang sama atau komputer lain. Dalam paradigma X 
       Window System, server berjalan di komputer yang menyediakan 
       perangkat display dan input. Server dapat memulai sesi dengan 
       sendirinya atau dapat terhubung ke manajer display yang meminta 
       untuk memulai sesi. Dalam kasus kedua, server X bertindak sebagai 
       klien telnet grafis sementara manajer tampilan bertindak seperti 
       server telnet: pengguna memulai program dari komputer yang 
       menjalankan manajer tampilan, sementara input dan output mereka 
       berlangsung di komputer tempat server (dan pengguna) duduk.

          Dalam X Window System, server X berjalan di komputer di depan 
       pengguna. Server X dapat terhubung ke manajer display yang berjalan 
       di komputer lain, memulai sesi yang terdiri dari berbagai program 
       yang berjalan pada komputer lain itu.

          Ketika server X terhubung ke manajer display, manajer tampilan
      tersebut menampilkan pengguna dengan layar masuk untuk memasukkan 
      nama pengguna dan kata sandi. Manajer tampilan dapat berjalan di 
      komputer yang sama dengan server X, sehingga mewujudkan Sistem X 
      Window berfungsi sebagai init, getty dan login pada terminal 
      mode karakter.

          X11 Release 3 memperkenalkan manajer display X pada bulan 
      Oktober 1988 dengan tujuan mendukung terminal X standalone yang 
      kemudian masuk ke pasar. Berbagai manajer display terus menggunakan 
      rutin untuk memberikan prompt login grafis pada komputer standalone 
      yang menjalankan X. X11R4 memperkenalkan XDMCP (Desember 1989) untuk 
      memperbaiki masalah dalam implementasi X11R3.
